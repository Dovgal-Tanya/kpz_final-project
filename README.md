# NoteReminder — Електронний записник з нагадуваннями

Фінальний проєкт з дисципліни "Конструювання програмного забезпечення".  
**Виконала:** Студентка групи ІПЗ-24-4, Довгаль Тетяна.

---

## Функціональність проєкту
Додаток призначений для зручного менеджменту повсякденних завдань, нотаток та часових сповіщень.

* **Управління записами (CRUD):** Створення, відображення в таблиці та видалення нотаток.
* **Категоризація та пріоритетність:** Розподіл завдань за категоріями (*Work, Personal, Study*) та рівнями важливості (*Low, Medium, High*).
* **Розумні нагадування:** Автоматичне відслідковування часу подій та миттєве візуальне сповіщення користувача на екрані при настанні дедлайну.
* **Пошук та фільтрація:** Швидкий пошук за ключовими словами у тексті, а також комбінована фільтрація за категорією та пріоритетом.
* **Поліморфне збереження:** Автоматичний експорт та імпорт усієї бази даних у локальний файл `notes_data.json` у форматі JSON.

## Інструкція для локального запуску
1. Клонуйте цей репозиторій:  
   `git clone https://github.com/Dovgal-Tanya/kpz_final-project.git`
2. Відкрийте файл рішення `NoteReminder.sln` у Visual Studio.
3. Перевірте наявність NuGet-пакету **Newtonsoft.Json** (якщо відсутній, встановіть через Package Manager Console: `Install-Package Newtonsoft.Json`).
4. Натисніть **F5** або кнопку **Start** для запуску додатку.

## Programming Principles (Принципи програмування)
У проєкті суворо дотримано ключових принципів написання чистого коду:

1. **Single Responsibility Principle (SRP / Принцип єдиної відповідальності):**  
   Кожен клас виконує лише одну чітку функцію. Наприклад, клас `ValidationHelper` займається суто перевіркою коректності введених даних, а `StorageService` відповідає виключно за збереження/завантаження файлів, не втручаючись у роботу UI.
2. **Open/Closed Principle (OCP / Принцип відкритості/закритості):**  
   Базовий клас `Note` відкритий для розширення, але закритий для модифікації. Ми легко створили два різні класи-наслідники `SimpleNote` та `ReminderNote`. Якщо у майбутньому знадобиться додати новий тип нотатки (наприклад, `VoiceNote`), ми зможемо це зробити без зміни коду існуючих класів.
3. **Don't Repeat Yourself (DRY / Не повторюйся):**  
   Логіка оновлення таблиці та очищення полів винесена в окремі методи форми (`RefreshNotesGrid`, `ClearFormInput`), що уникає дублювання коду в обробниках подій кнопок.
4. **KISS (Keep It Simple, Stupid / Будь простішим):**  
   Для збереження даних замість розгортання важковагових локальних СКБД використано просту та швидку серіалізацію у файл JSON за допомогою рішень бібліотеки Newtonsoft.Json.
5. **Principle of Least Knowledge (Закон Деметри):**  
   Компоненти UI не знають, як саме влаштовано збереження даних чи внутрішня фільтрація колекцій. Головна форма спілкується виключно з високорівневим менеджером `NoteManager`.

## Design Patterns (Патерни проєктування)
В архітектурі реалізовано 3 фундаментальні патерни:

1. **Singleton (Одинак):**  
   *Файл:* `Common/ConfigurationManager.cs`  
   *Навіщо:* Гарантує існування лише одного екземпляра класу конфігурацій для всього додатку, що запобігає конфліктам під час визначення глобальних шляхів до файлів збереження.
2. **Factory Method (Фабричний метод):**  
   *Файл:* `Services/NoteFactory.cs`  
   *Навіщо:* Інкапсулює складну логіку створення об'єктів нотаток (`SimpleNote` чи `ReminderNote`) залежно від обраного типу в інтерфейсі, звільняючи від цієї відповідальності код форми.
3. **Observer (Спостерігач):**  
   *Файл:* `Services/ReminderScheduler.cs`  
   *Навіщо:* Реалізований через механізм подій C# (`event Action`). Клас планувальника відслідковує час у фоні й "сповіщає" підписану на нього графічну форму про те, що настав час вивести повідомлення про дедлайн для користувача.

## Refactoring Techniques (Техніки рефакторингу)
Під час розробки застосовано щонайменше 5 технік рефакторингу для боротьби із "запахами коду" (code smells):

1. **Extract Method (Виділення методу):** Велика логіка валідації текстових полів була виділена з обробника кліку кнопки додавання у чистий окремий метод `ValidateNoteInput` в утилітарному класі.
2. **Move Method (Переміщення методу):** Логіка фільтрації та пошуку за ключовими словами спочатку була частиною UI форми, але згодом перенесена у відповідний сервісний клас `NoteManager.cs` задля розвантаження UI-шару.
3. **Rename Variable / Method (Перейменування):** Усі змінні та методи отримали зрозумілі бізнес-назви (наприклад, замість абстрактного `d` використано `currentMoment`, замість `txt` — `jsonString`).
4. **Decompose Conditional (Декомпозиція умовних виразів):** Складні перевірки прапорців та типів об'єктів (наприклад, визначення, чи є нотатка нагадуванням) винесені у зрозумілі конструкції з ключовим словом `as`, що покращило читабельність коду.
5. **Remove Dead Code (Видалення мертвого коду):** З усіх файлів проєкту повністю прибрано невикористовувані системні директиви `using`, пусті методи-обробники, які генерує Visual Studio за замовчуванням, та тестові коментарі.
